Dataset Selection and Constraints

This guide will help you navigate the process of selecting an appropriate dataset for your project.

  • Criteria for choosing a dataset:
    You are only allowed to use datasets that adhere to the following properties:

    • Number of Instances: 500 or more
    • Number of Features: Between 10 and 100
    • Data Type of Features: Numerical only. The features in the dataset should consist of only real and/or integer values. This restriction simplifies data preprocessing and ensures compatibility with many machine learning algorithms.
    • Ensure that your selected dataset is suitable for classification tasks. This means that it must have a discrete target variable and not one with continuous numerical values.
    • Exclusion: Please note that you must specifically exclude the Wine Quality dataset from your selection, as it is commonly used in many standard machine learning tutorials, and the goal is to encourage exploration of less common datasets.

  • Applying filters:
    You can use the filter options of the UCI Library to narrow down the selection of viable datasets. To search for datasets with numerical features, you can use the "Multivariate" and "Tabular" filter options in the "Data Type" category, as well as the "Numerical" option in the "Feature Type" category.
    However, while these filters are useful, they may not capture all datasets that meet the criteria, hence an additional manual search may be warranted.

  • Manual verification: It is essential that you manually verify that your chosen dataset adheres to the project's requirements. This includes confirming the dataset’s suitability based on size, feature set, and target variable.

  • Multiclass consideration: If selecting a multiclass dataset, you may limit it to two classes as long as at least 500 instances remain. This can be done by removing other classes or by merging classes in a way that makes sense.

    For example, in the Heart Disease dataset (the example below), the target variable of most data files has values from 0 (no presence) to 4. To convert this into a dataset suitable for binary classification, you could combine all values 1 through 4 into one class by replacing all values greater than 0 with 1, representing "presence," while keeping 0 as "absence."

    If you choose to merge classes, ensure that the resulting dataset still retains meaningful information and supports the goals of your experiment.

  • Avoid multilabel datasets: Multilabel datasets are strongly discouraged and should ideally be avoided, due to the imposed additional analytical challenges.

Example Dataset: Heart Disease Detection

The following dataset from the UCI Library is an example that fulfills the constraints for this assignment: https://archive.ics.uci.edu/ml/datasets/heart+disease

This dataset contains data on heart disease diagnosis. The processed.<country>.data files contain 14 columns, including the target variable that indicates the presence (or absence) of heart disease in the patient. Missing values in this dataset are specified with ? in the data files.

  • Number of Instances: Although the dataset overview shows 303 instances (referring to the Cleveland data file), the full dataset has 920 instances, including data from Cleveland (303), Hungary (294), Switzerland (123) and VA Long Beach (200). This meets the minimum requirement of 500 instances, but it highlights the fact that dataset filters may not always work as expected.
  • Number of Features: The dataset contains 13 attributes, which falls within the acceptable range of 10 to 100 features.
  • Data Type of Features: All features in the dataset are numerical.
  • Classification Suitability: The target variable for this dataset is discrete, with the objective being to determine whether a patient has heart disease, based on the provided features. There are no continuous target variables, ensuring it aligns with classification requirements.

Class DatasetPreprocessor (4P)

The class DatasetPreprocessor, at its core, is designed to facilitate the conversion of your chosen dataset from a .zip archive to a single .csv file. The following requirements should be met:

  • Ensure all data instances are consolidated into one .csv file.
  • Assess and handle missing values. This could involve strategies such as imputation, deletion, or data augmentation.
  • Get rid of unwanted classes if you are using a multiclass dataset, but do not want to do a multiclass classification.
  • Keep in mind that not all files within the .zip archive may be required for your classification task. To avoid unnecessary processing and confusion, read the dataset's description carefully to understand the purpose of each file and determine which ones are essential for your experiments.

It is up to you how you ensure that all constraints are met. Feel free to implement functionality beyond the required methods.
Note: Your implementation of the DatasetPreprocessor only has to work with your chosen dataset, not with any other datasets.

Class SimpleBaselineClassifier (3P)

The SimpleBaselineClassifier makes trivial predictions that do not use any information from the features in the training data. Such a baseline is useful for determining whether (and by how much) the predictive performance of other, more complex classifiers is better than what could be achieved with the most naive approach.
Note: The SimpleBaselineClassifier implements some of the functionality of sklearn's DummyClassifier. You can use the DummyClassifier as a reference for more information on the strategies.

You should implement the following strategies:

  • most_frequent: This strategy always predicts the most frequent value of the target variable in the training data.
  • uniform: This strategy predicts the value of the target variable at random, chosen from the set of unique values that occur in the (target variable of the) training data. You can use numpy.random.RandomState for reproducibility.
  • constant: This strategy always predicts the same value.

You may be required to present the code of your SimpleBaselineClassifier class during your presentation/code review.

Required Method: __init__

  • Functionality:
    This method stores the value of the parameters in public instance attributes of identical names.

  • Parameters:

    Name Type Default Value Description
    strategy str 'most_frequent' The strategy that the SimpleBaselineClassifier should use to make simple predictions. Possible values: most_frequent, uniform, constant.
    random_state int None Used for reproducible output when using the strategy uniform.
    constant int or str None If the strategy constant is used, this parameter tells the SimpleBaselineClassifier which value it should (always) predict. The value of the constant has to be present in the (target values of the) training data.

Required Method: __repr__

  • Functionality: Returns the formal string representation of the class. The method returns the following string:
"SimpleBaselineClassifier(strategy=<strategy>, random_state=<random_state>, constant=<constant>)"
  • Return:
    Type Description
    str Formal string representation of the class

Required Method: fit

  • Functionality:
    This method stores the method's parameters in non-public instance attributes.

  • Parameters:

    Name Type Shape Description
    X_train numpy.ndarray (<num_train_samples>,<num_features>) The feature matrix of the training set.
    y_train numpy.ndarray (<num_train_samples>,) The target vector of the training set.

Required Method: predict

  • Functionality:
    This method predicts the values of the target variable for the provided feature matrix of the test set (parameter X_test) according to the chosen strategy (that the __init__ stored in an instance attribute).

  • Parameters:

    Name Type Shape Description
    X_test numpy.ndarray (<num_test_samples>,<num_features>) The feature matrix of the test set.

  • Return:

    Type Shape Description
    numpy.ndarray (<num_test_samples>,) The predicted values of the target variable.

Classification Experiments (14P)

When you have implemented the required classes above, you can start with your classification experiments. For this, you will implement functionality for supervised learning. In your classification experiments, you will train different classifiers on your training data and use them to predict the values of the target variable of the samples contained in your test data.

Some classifiers will predict these values better than others. Once a classifier is trained, it can be used to make predictions on new data. A test set, i.e. a set of samples which were not used for training but also have known labels, allows to estimate the classifier’s predictive performance on unseen data. For this task, out of the wide variety of evaluation metrics applicable to different classification tasks, you must calculate and report at least the following:

  • Accuracy
  • Precision
  • Recall
  • F1 Score

You are allowed to use scikit-learn for calculating the evaluation metrics.

You will design any code that you write for the experiments yourself. Plan and discuss the design of the code with your other group members before you implement it together. In your technical report and your presentation/code review, you will have to present the design of your code along with the setup and results of your classification experiments.

Instructions for Conducting the Experiments

The following steps are required for your classification experiments (in addition to designing and implementing any code that you need for the experiments).

Preprocessing

Before diving into the experiments, proper preprocessing of your chosen dataset is essential. The specific preprocessing steps vary based on the dataset, but here are some general guidelines to follow:

  1. Choose a dataset based on the requirements given in the Dataset section.
  2. Preprocess your dataset using your DatasetPreprocessor class. Your preprocessed data should result in a .csv file that adheres to the following criteria:
    • Ensure that all data instances that you will use for classification are consolidated into one .csv file.
    • Assess and handle missing values. This could involve strategies such as imputation, deletion, or data augmentation.
    • In case you are using a multiclass dataset but don't want to do a multiclass classification, convert the dataset in a way that makes it suitable for binary classification, according to your chosen approach (e.g., delete or merge classes).

After preprocessing, your dataset should be in an optimal form for training the machine learning models you will be testing.

  1. Bonus: Visualize the dataset. For additional information, refer to the bonus points section below.

Classification and Visualization of the Results

  1. Choose a set of classifiers that you want to use for predicting the target variable.
    • Choose at least four supervised classifiers from scikit-learn: Look through the classifiers available in scikit-learn and choose at least four (suitable) classifiers that you want to use.
  2. Choose a suitable baseline to compare the performance of the classifiers to. For the baseline, you should use one of the strategies of your SimpleBaselineClassifier.
  3. Think about what percentage of your data you want to use as training data and what percentage you want to use as test data.
    • Also think about which features you want to use.
  4. Train your classifiers on the training data and use them to predict the values of the target variable of the samples in the test data.
  5. Evaluate the results. Which classifier performed best? Which one performed the worst? Did all classifiers beat the performance of your chosen baseline?
    • Use (at least) the mandatory evaluation metrics for your evaluation. You are allowed to use scikit-learn for calculating the evaluation metrics.
  6. Visualize the results of your evaluation. At a minimum, (1) create plots that show the confusion matrices of all different classifiers and (2) create bar charts that show all your mandatory and chosen evaluation metrics for all different classifiers. Also include the baseline in your plots. Work with your preferred plotting library (matplotlib or plotly). You can see examples for such plots below.
    • Each plot that you design should have, as a minimum:
      • A title
      • Axis labels
      • Axis tick labels
      • An appropriate scale for the axes
